## What is Agencia ESA?

Agencia ESA (`AgenciaESA`) is a Java 24 / Maven console application built for the 2nd-year DAM (Development of Multiplatform Applications) exam at IES Virgen de la Paloma. It manages ESA space agency data stored in a MySQL database, and supports three core operations:

<Steps>
  <Step title="Export unassigned crew to CSV">
    Retrieve every `Tripulante` who has no active mission assignment and write their details to a pipe-delimited CSV file.
  </Step>
  <Step title="Process XML operations file">
    Parse `operaciones_esa.xml` using Jackson's `XmlMapper`, update existing missions in the database, and insert or update crew members referenced in the file.
  </Step>
  <Step title="Query missions by year">
    Retrieve and print to console all missions that started in a given year, including the spacecraft and destination celestial body.
  </Step>
</Steps>

## Key features

<CardGroup cols={2}>
  <Card title="JDBC / MySQL" icon="database">
    All persistence uses raw JDBC with `PreparedStatement` and the DAO pattern. No ORM required.
  </Card>
  <Card title="DAO pattern" icon="layer-group">
    Clean separation between business logic and data access through DAO interfaces and JDBC implementations.
  </Card>
  <Card title="XML processing" icon="file-code">
    Jackson `XmlMapper` parses ESA operation feeds and applies database updates transactionally.
  </Card>
  <Card title="CSV export" icon="file-csv">
    Configurable separator and header generate pipe-delimited crew reports with a single method call.
  </Card>
  <Card title="Properties-based config" icon="gear">
    All JDBC credentials, file paths, and CSV settings live in `aplicacion.properties` — no hard-coded values.
  </Card>
  <Card title="Typed exceptions" icon="triangle-exclamation">
    `AgenciaEsaException` wraps SQL and I/O errors with an error code and the originating class for precise diagnostics.
  </Card>
</CardGroup>

## Data model overview

The application manages five core entities persisted in MySQL:

| Entity | Table | Description |
|---|---|---|
| `Mision` | `T_MISION` | A space mission with dates, status, budget, and objective |
| `Nave` | `T_NAVE` | A spacecraft identified by a code, type, and crew capacity |
| `CuerpoCeleste` | `T_CUERPO_CELESTE` | A destination body (planet, moon, etc.) with surface gravity |
| `Tripulante` | `T_TRIPULANTE` | A crew member with ESA ID, specialty, and experience level |
| `TripulanteMision` | `T_TRIPULANTE_MISION` | Many-to-many link between crew members and missions |

```

├── configuracion/
│   └── aplicacion.properties       # JDBC, CSV, and XML paths

│   ├── operaciones_esa.xml         # ESA operations feed (input)
│   └── tripulantes_sinmision.csv   # Crew export output
├── src/main/java/eu/esa/gemis/
│   ├── AplicacionAgenciaESA.java   # Entry point + main menu
│   ├── dao/                        # DAO interfaces
│   ├── dao/impl/                   # JDBC implementations
│   ├── excepcion/                  # Custom exception
│   ├── gestor/                     # JDBC connection + config reader
│   ├── procesos/                   # CSV and XML processors
│   └── vo/                         # Value objects (POJOs)
└── pom.xml
```
