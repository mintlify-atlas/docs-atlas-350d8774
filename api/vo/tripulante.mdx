# Tripulante

**Package:** `eu.esa.gemis.vo`  
**DB table:** `T_TRIPULANTE`

The `Tripulante` value object represents an ESA crew member (astronaut or mission specialist). It carries two distinct numeric identifiers â€” a database surrogate key and a separate ESA business identifier â€” which serve different purposes.

<Info>
  `Tripulante` is a plain Java bean with no persistence logic of its own. DAO classes handle all SQL interaction against `T_TRIPULANTE`.
</Info>

<ResponseField name="numeroTripulante" type="int" required>

## The two identifiers explained

<Warning>
  `numeroTripulante` and `identificadorEsa` are **not interchangeable**.
</Warning>

| Identifier | Type | Origin | Purpose |
|---|---|---|---|
| `numeroTripulante` | `int` | DB auto-increment (`T_TRIPULANTE` PK) | Internal row ID; used in SQL JOINs and as FK in mission-crew tables. |
| `identificadorEsa` | `int` | ESA HR system | Organisation-wide business ID; used for lookups across ESA systems and in reports. |

When inserting a new `Tripulante`, omit `numeroTripulante` (leave it `0` or do not include it in the `INSERT` column list) and let the database assign it. `identificadorEsa` must always be provided explicitly.

## Construction and usage example

```java

import java.time.LocalDate;

// New crew member â€” do NOT set numeroTripulante; DB assigns it on INSERT

tripulante.setNombre("Elena");
tripulante.setApellidos("GarcĂ­a LĂłpez");

tripulante.setNivelExperiencia(4);
tripulante.setEspecialidad("PILOTO");

// After INSERT, the DAO sets the generated key:

// Tripulante [numeroTripulante=0, nombre=Elena, ...]
```

<Tip>
  When searching for a crew member from an external ESA context, use `identificadorEsa`. When building SQL JOINs to mission-crew assignment tables within GEMIS, use `numeroTripulante`.
</Tip>
