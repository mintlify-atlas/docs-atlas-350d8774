## Overview

`INaveDAO` defines the persistence contract for `Nave` (spacecraft) entities. The JDBC implementation queries the `T_NAVE` table.

| Property | Value |
|---|---|
| Interface | `eu.esa.gemis.dao.INaveDAO` |
| Primary table | `T_NAVE` |
| Checked exception | `eu.esa.gemis.excepcion.AgenciaEsaException` |

<Note>
  The method signatures below are derived from the interface source file and standard DAO conventions used throughout the Agencia ESA codebase. The `T_NAVE` table columns include `codigo` (primary key), `nombre`, `capacidad_tripulacion`, `tipo`, and `estado`. Refer to the source file `eu/esa/gemis/dao/INaveDAO.java` for the authoritative list.
</Note>

## `T_NAVE` table structure

| Column | Type | Description |
|---|---|---|
| `codigo` | `VARCHAR` | Primary key — spacecraft code |
| `nombre` | `VARCHAR` | Spacecraft name |
| `capacidad_tripulacion` | `INT` | Maximum crew capacity |
| `tipo` | `VARCHAR` | Spacecraft type/class |
| `estado` | `VARCHAR` | Operational status |

## Expected methods

### `obtenerNaves`

Retrieves all spacecraft records from `T_NAVE`.

**Signature**

```java
List<Nave> obtenerNaves() throws AgenciaEsaException;
```

#### Return value

A `List<Nave>` containing all spacecraft. Returns an empty list when the table has no rows.

### `obtenerNavePorCodigo`

Retrieves a single spacecraft by its primary key code.

**Signature**


#### Parameters

### `insertarNave`

Persists a new spacecraft record in `T_NAVE`.

**Signature**

```java
void insertarNave(Nave nave) throws AgenciaEsaException;
```

#### Parameters

<ParamField path="nave" type="Nave" required>
  Fully populated `Nave` value object. The `codigo` field must be unique.
  <Expandable title="Fields">
    <ResponseField name="codigo" type="String">
      Spacecraft code (primary key). Must not already exist in `T_NAVE`.
    </ResponseField>
    <ResponseField name="nombre" type="String">
      Spacecraft name.
    </ResponseField>
    <ResponseField name="capacidadTripulacion" type="int">
      Maximum crew capacity.
    </ResponseField>
    <ResponseField name="tipo" type="String">
      Spacecraft type or class.
    </ResponseField>
    <ResponseField name="estado" type="String">
      Operational status.
    </ResponseField>
  </Expandable>
</ParamField>

#### Return value

`void`. The method completes silently on success.

#### Exceptions

<ResponseField name="AgenciaEsaException" type="checked exception">

  <Expandable title="Error codes">
