## Overview

`ICuerpoCelesteDAO` defines the persistence contract for `CuerpoCeleste` (celestial body) entities. The JDBC implementation queries the `T_CUERPO_CELESTE` table, which is also referenced by `T_MISION` via the `id_cuerpo_celeste` foreign key.

| Property | Value |
|---|---|
| Interface | `eu.esa.gemis.dao.ICuerpoCelesteDAO` |
| Primary table | `T_CUERPO_CELESTE` |
| Checked exception | `eu.esa.gemis.excepcion.AgenciaEsaException` |

<Note>
  The method signatures below are derived from the interface source file and standard DAO conventions used throughout the Agencia ESA codebase. The `T_CUERPO_CELESTE` table columns include `identificador` (primary key), `nombre`, `tipo`, and `distancia_tierra_km`. Refer to the source file `eu/esa/gemis/dao/ICuerpoCelesteDAO.java` for the authoritative list.
</Note>

| Column | Type | Description |
|---|---|---|
| `identificador` | `INT` | Primary key â€” auto-generated celestial body ID |
| `nombre` | `VARCHAR` | Unique celestial body name |
| `tipo` | `VARCHAR` | Classification (e.g. planet, moon, asteroid) |

<Info>
  The `nombre` column is used as a human-readable natural key in several queries across the system. For example, `MisionDaoJDBC` resolves `id_cuerpo_celeste` from `nombre` using a correlated subquery rather than requiring callers to know the numeric identifier.
</Info>

## Relationship to `IMisionDAO`

`T_CUERPO_CELESTE` is referenced from `T_MISION` via a foreign key on `id_cuerpo_celeste`. When `MisionDaoJDBC` updates a mission, it resolves the celestial body's numeric identifier from its name using the following subquery:

```sql
SELECT tcc.identificador
FROM   T_CUERPO_CELESTE tcc
WHERE  tcc.nombre = ?
```

This means the `nombre` value stored on a `Mision` value object must exactly match a row in `T_CUERPO_CELESTE`.
